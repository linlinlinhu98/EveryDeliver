## 一、项目总览

**项目名称**：求职者半自动投递辅助平台

**核心定位**：通过 AI 智能筛选与适配提升投递效率

**解决的痛点**：
1. 多平台搜寻匹配岗位 + 简历定制 + 投递耗时巨大
2. 不确定简历与岗位的匹配度，不知道如何优化提升命中率

## 三、关键设计决策汇总

### 决策 1：模式系统（询问 vs 自动）

- **询问模式**：每处改动弹窗询问
- **自动模式**：按改动风险分级
  - 措辞优化（🟢）→ 直接应用
  - 侧重点调整（🟡）→ 直接应用
  - GitHub 化用（🟠）→ 弹轻提示
  - 新增关键词（🔴）→ 仍弹窗
  - 新增经历（🔴）→ 仍弹窗 + 强制确认
- **模式可随时切换**
- AI 根据用户行为主动建议切换

### 决策 2：新增内容的处理

**不再是门控**（不问"是否做过"），**改为赋能**：
- 展示新增内容
- 标注参考来源（GitHub 链接）
- **自动生成面试准备建议**（基于新增内容可能被问到的题目）
- 一键加入面试准备清单

### 决策 3：跨段改动处理

- 完全独立，无联动
- 一个改动可改，另一个可不改
- 用户可单独回退任意一处

### 决策 4：低匹配度提示（最终版）

```
匹配度仅 28%，AI 优化提升空间有限。

[💡 需要我帮您提出更多优化建议吗]
[⏹️ 就这样，不再继续改动]
[🚪 考虑放弃该岗位]
```

### 决策 5：面试准备新增"精神风貌"模块

JD 中的模糊品质要求（"抗压能力强""有创业精神"）：
- **不解析**进结构化数据
- **进入面试准备清单**，生成具体准备策略

### 决策 6：技能分类两层

- **直接相关技能**：JD 中明确提到的，权重 × 2.0
- **通用技能**：基础工具/软技能，权重 × 0.5

### 决策 7：JD 生命周期

```
[导入] → active
         ├─ 30 天未更新 → expiring_soon
         ├─ 重抓内容有变 → 覆盖（保留旧快照）
         └─ 重抓发现停招 → expired + 通知用户
```

### 决策 8：模型策略（国产 + 分级）

| 任务 | 模型 | 理由 |
|------|------|------|
| JD 解析 | DeepSeek-V3 | 便宜稳定 |
| 简历优化 | Qwen-Plus | 语义改写要求高 |
| 匹配度评估 | DeepSeek-V3 | 公式化任务 |
| 改动说明 | DeepSeek-V3 | 模板化生成 |
| 面试题生成 | Qwen-Max | 质量优先 |
| 面试准备建议 | Qwen-Plus | 平衡 |

**成本估算**：约 ¥0.86/月/用户（用 Claude 的 1/15）

### 决策 9：多端分工

| 端 | 定位 | 关键能力 |
|---|------|---------|
| **桌面端** | 主战场 | 完整文件访问 + 解析 + 编辑 + AI 优化 |
| **浏览器插件** | 触达 | JD 抓取 + 表单填充（用户最终手动提交）|
| **小程序** | 随身 | 查看 + 记录 + 下载简历 + 跳转投递 |

### 决策 10：用户数据策略

- 默认私有
- 训练数据 opt-in（弹窗告知贡献内容，可随时关闭）
- 不存密码 / cookies / 自动投递

## 十一、浏览器插件架构关键决策

| 维度 | 决策 |

| 同步策略 | 首次全量 + 后续增量 |
| 隐私保护 | 首次启用弹窗告知 + 随时可关闭 |
