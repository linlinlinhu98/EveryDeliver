### A5 黑名单标签来源（4 种自动检测机制）

```
黑名单标签来源
├── 1. JD 文本分析（关键词匹配）★ 主要 — 零成本
│   ├── 工作时间类：大小周 / 单休 / 996 / ICU / 加班严重
│   ├── 工作性质类：外包 / 派遣 / 驻场
│   ├── 风险信号类：试用期长 / 考核淘汰 / 付费培训 / 股权激励
│   └── 模糊承诺类：薪资面议 / 不加班（反讽检测）
├── 2. 公司工商信息查询（Phase 2）
│   ├── 注册资本 < 100 万 → 提示小公司风险
│   ├── 成立时间 < 1 年 → 提示新公司风险
│   ├── 经营异常/失信 → 提示法律风险
│   ├── 实缴资本为 0 → 提示皮包公司
│   ├── 社保人数 < 10 → 提示小团队
│   └── 行业代码属外包/派遣 → 提示外包性质
├── 3. 行业分类默认警告 — 零成本
│   └── 内置 7 类高风险行业清单（教培(P2P)/房地产/区块链/外包/直播带货/跨境电商），用户可一键关闭
└── 4. 用户历史行为推断 — 零成本
    ├── 用户拒绝过该公司 → 自动加入黑名单
    ├── 用户多次拒绝类似 JD → 自动识别模式
    └── 用户手动加过"外包"→ 后续所有"外包"自动加入
```

### 自动检测关键决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 置信度阈值 | ≥3 个信号才自动加入 | 平衡自动化与误判 |
| 工商 API | Phase 2 | MVP 零外部依赖 |
| 行业默认警告 | 默认开 + 必读 + 可关闭 | 保护用户但不强制 |
| 信号矛盾时 | 不自动加入，标记"信息不足" | 把决定权留给人 |
| 社区标签 | Phase 2 | P2 必须做 |

### 功能清单

| B1 | 手动职位导入 | 粘贴/录入职位信息（公司/标题/JD 摘要/链接） | P0 |
| B2 | 浏览器插件抓取职位 | BOSS 直聘 + 猎聘硬编码适配 + 长尾 AI 兜底 | P0 |
| B3 | 三层过滤 | ① 黑名单命中→丢弃；② 历史重复→移入重复列表；③ 薪资/城市不匹配→标记不删除 | P0 |
| B4 | 可勾选职位表格 | 含匹配度/链接/位置/来源平台 | P0 |
| B5 | 职位详情跳转 | 新标签页打开目标平台 | P0 |

### 三层过滤流程

```
用户导入 JD → [黑名单命中？]→是→丢弃
                       ↓否
                  [历史重复？]→是→移入重复列表+提示原因
                       ↓否
                  [薪资/城市不匹配？]→是→标记警告（不删除）
                       ↓否
                  进入可勾选职位表格
```

### 浏览器插件合规边界

**首次启用告知弹窗（必须显示）**：
- 说明工作方式（读取页面内容、提取职位信息、自动填充表单）
- 合规风险提示（可能违反目标平台协议、可能导致账号受限或封禁）
- 隐私保护说明（不读取聊天、不修改账号、不存密码、数据本地加密）
- 用户确认 ☑ 我已了解上述风险，自愿启用本插件

**法律配套**：
- 用户协议明确写"插件功能可能违反第三方平台协议，风险自担"
- 隐私政策明确写"插件读取的页面内容不上传到云端训练"
- 每次启动时显示简短风险提示（首次后每月一次）

**技术安全姿势**：
- 模拟真实用户操作（随机延迟、自然滚动）
- 不批量抓取（每平台每天上限，10 秒间隔）
- 夜间 0-7 点自动暂停
- 验证码检测到→立即停止+提示用户

| C5 | 人工复核确认 | 询问模式 + 自动模式可随时切换 | P0 |

### 匹配度评估公式

```
总分 = 硬性匹配(60%) + 软性匹配(30%) + 隐性匹配(10%)

硬性匹配（60%）：
├── 学历要求符合
├── 经验年限符合
├── 必备技能命中
└── 技能分类权重：直接相关技能 ×2.0 / 通用技能 ×0.5

软性匹配（30%）：
├── 项目经历相关性
├── 行业经验重合度
└── 职责语义相似度

隐性匹配（10%）：
├── 技术栈趋势契合度
└── 职业路径合理性
```

### 模式系统（询问 vs 自动）

| 改动类型 | 风险等级 | 询问模式 | 自动模式 |
|----------|---------|---------|---------|
| 措辞优化 | 🟢 | 直接应用 | 直接应用 |
| 侧重点调整 | 🟡 | 弹窗确认 | 直接应用 |
| GitHub 化用 | 🟠 | 弹窗确认 | 弹轻提示 |
| 新增关键词 | 🔴 | 弹窗确认 | 弹窗确认 |
| 新增经历 | 🔴 | 弹窗确认 | 弹窗+强制确认 |

### 改动高亮三色系统

```
【黄色】已有内容改写 — 措辞优化、侧重点调整
【橙色】外部参考内容 — GitHub 化用、模板引用
【红色】新增内容 — 新技能、新项目经历
```

每处改动附带：
- 改动原因说明
- 参考来源（如有）
- 面试准备建议（新增内容自动生成）

### 低匹配度提示

```
匹配度仅 28%，AI 优化提升空间有限。

[💡 需要我帮您提出更多优化建议吗]
[⏹️ 就这样，不再继续改动]
[🚪 考虑放弃该岗位]
```

### 新增内容的面试赋能

- 面试可能被追问的问题

## 模块 G：JD Parser

### 功能清单

| # | 功能 | 说明 | 阶段 |
|---|------|------|------|
| G1 | 数据模型标准化 | 4 层结构 + 元数据 | P0 |
| G2 | 技能词表 | 直接相关技能 vs 通用技能分层 | P0 |
| G3 | 三层解析架构 | 规则 + LLM + 用户反馈 | P0 |
| G4 | 置信度机制 | 高/中/低三级处理 | P0 |
| G5 | 失败兜底 | 降级到规则解析 + 用户补充 | P0 |
| G6 | 生命周期管理 | active / expiring / expired | P0 |

### 4 层数据模型

```
Layer 1 — 硬性要求（结构化）
├── 学历：[本科/硕士/博士/不限]
├── 经验年限：[0-1/1-3/3-5/5-10/10+]
├── 薪资范围：[min, max]
├── 工作地点：[城市/远程/混合]
└── 语言要求：[中文/英文/不限]

Layer 2 — 技能要求（标签化）
├── 直接相关技能（权重 ×2.0）：JD 明确提到的技术栈
└── 通用技能（权重 ×0.5）：基础工具/软技能

Layer 3 — 职责描述（语义化）
├── 核心职责摘要（1-2 句）
└── 关键词提取

Layer 4 — 隐性要求（推断）
├── 团队规模推断
├── 技术栈趋势
└── 岗位紧急度
```

### 三层解析架构

```
JD 文本输入
    ↓
[Layer 1] 规则解析 ★ 主力
    ├── 覆盖平台：BOSS 直聘 / 猎聘
    ├── 处理字段：薪资、地点、学历、经验年限、公司名
    ├── 覆盖率目标：薪资 95%、地点 90%、学历 85%、经验 80%、公司名 90%
    └── 置信度 < 0.7 → 降级到 LLM
    ↓
[Layer 2] LLM 解析 ★ 兜底
    ├── 触发条件：规则置信度 < 0.7 / 必填字段缺失 / JD < 50 字或 > 5000 字
    ├── 处理：技能提取、职责总结、JD 语义匹配
    ├── 模型：DeepSeek-V3
    └── 成本：¥0.004/JD
    ↓
[Layer 3] 用户反馈 ★ 持续优化
    ├── 用户修正 → 记录修正案例
    ├── 同 JD 不重复解析（缓存 key = URL hash）
    ├── 积累 100 条 → 作为 few-shot 示例更新 prompt
    └── 高频修正模式 → 人工审核后提取为新规则
```

### 平台覆盖与降级策略

| 平台 | 规则解析字段 | 必走 LLM 字段 | 降级策略 |
|------|-------------|-------------|---------|
| BOSS 直聘 | 薪资/地点/学历/经验/公司名 | 技能提取/职责总结/语义匹配 | 规则失败 → LLM |
| 猎聘 | 薪资/地点/学历/经验/公司名 | 同上 | 同上 |
| 微信公众号 | 几乎全部走 LLM | 全部 | 规则跳过 → LLM |
| 公司官网 | 几乎全部走 LLM | 全部 | 规则跳过 → LLM |

### JD 生命周期

```
[导入] → active（活跃）
           ├─ 30 天未更新 → expiring_soon（即将过期）
           ├─ 重抓内容有变 → 覆盖（保留旧快照）
           └─ 重抓发现停招 → expired（已过期）+ 通知用户
```

| 端 | 定位 | 编辑权限 | 关键能力 |
|----|------|---------|---------|
| 桌面端 | 主战场 | ✅ 独享编辑权 | 完整文件访问 + 解析 + 编辑 + AI 优化 |
| 浏览器插件 | 触达 | ❌ 只读 | JD 抓取 + 表单自动填充（用户手动提交） |
| 微信小程序 | 随身 | ❌ 只读 | 查看 + 记录 + 下载简历 + 跳转投递 |

### 多端编辑策略

```
桌面端：独享编辑权（强锁 5 分钟 TTL）
小程序：只读 + 下载
浏览器插件：只读 + 同步

锁过期后：
├── 编辑端网络恢复 → 自动续锁
└── 其他端尝试编辑 → 提示"桌面端正在编辑中"
```

### 浏览器插件关键决策

| 维度 | 决策 |
|------|------|
| 平台适配 | 混合（BOSS/猎聘硬编码 + 长尾 AI 兜底） |
| MVP 覆盖 | BOSS + 猎聘 + Generic |
| 数据流 | 自动入库 + 桌面端批量审核 |
| 表单填充 | 半自动（预览→确认→用户手动提交） |
| 桌面端依赖 | 基础功能独立 + LLM 走桌面端 |
| 反爬策略 | 借助用户登录态 |
| 多账号 | 支持（同平台多账号） |
| 状态同步 | 手动为主 + 进入平台自动同步 |

## 关键设计决策汇总

| # | 决策 | 内容 |
|---|------|------|
| 1 | 模式系统 | 询问/自动双模式，按改动风险 5 级分级处理 |
| 2 | 新增内容处理 | 不再是门控，改为面试准备赋能 |
| 3 | 跨段改动 | 完全独立，无联动，可单独回退 |
| 4 | 低匹配度提示 | 三选项：继续优化/就这样/放弃 |
| 5 | 精神风貌模块 | JD 模糊品质要求不进结构化，进面试准备 |
| 6 | 技能分类 | 直接相关(×2.0) + 通用(×0.5) |
| 7 | JD 生命周期 | active→expiring→expired |
| 8 | 模型策略 | DeepSeek-V3 + Qwen-Plus/Max，LLM 代理不暴露 Key |
| 9 | 多端分工 | 桌面主战场 / 插件触达 / 小程序随身 |
| 10 | 用户数据 | 默认私有 / 训练 opt-in / 不存密码 |

### 模型分配策略

| 任务 | 模型 | 理由 |
|------|------|------|
| JD 解析 | DeepSeek-V3 | 便宜稳定 |
| 简历优化 | Qwen-Plus | 语义改写要求高 |
| 匹配度评估 | DeepSeek-V3 | 公式化任务 |
| 改动说明 | DeepSeek-V3 | 模板化生成 |
| 面试题生成 | Qwen-Max | 质量优先 |
| 面试准备建议 | Qwen-Plus | 平衡 |

**成本估算**：约 ¥0.86/月/用户（假设月投递 20 个岗位，每岗位触发匹配评估 + 1 次简历优化，10% 触发面试题生成）
