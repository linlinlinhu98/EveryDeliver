### 何时使用多 Agent 并行 vs 单 Agent 串行

| 场景 | 策略 | 工具 |
|------|------|------|
| 前后端独立任务 | 并行 2 个 Agent | Agent × 2 |
| 多文件同类改动 | Workflow pipeline | Workflow |
| 需要先设计再实现 | Plan Agent → 串行实现 | Plan Agent |
| 代码审查 | 单独 Agent 审查 | /code-review |
| 安全敏感模块 | 单独 Agent 审查 | /security-review |

### Phase 2：JD Parser

```
我现在处于 EveryDeliver 项目 Phase 2，需要实现 JD Parser 模块。

你已经读取过 docs/DESIGN.md 和 docs/ARCHITECTURE.md，请严格遵循其中的设计。

【Phase 2 目标】
BOSS 直聘 / 猎聘 JD 能被正确解析为 4 层结构化数据。

【你要做的事】

任务 1 — JD 数据模型迁移
  - 如果 Phase 0 未创建相关表，创建 job_positions / jd_parse_results / jd_parse_snapshots 表

任务 2 — BOSS 直聘规则解析器（⭐ 主力）
  - 根据 BOSS 直聘页面 DOM 结构写 CSS 选择器
  - 提取字段：薪资/地点/学历/经验年限/公司名
  - 注意 BOSS 的薪资格式是"15K-25K"形式
  - 覆盖率目标：薪资95%/地点90%/学历85%/经验80%/公司名90%

任务 3 — 猎聘规则解析器
  - 同上，适配猎聘 DOM 结构
  - 猎聘薪资格式不同，需适配

任务 4 — LLM 解析器（兜底）
  - Edge Function：POST /functions/v1/parse-jd
  - 接收原始 JD 文本 → DeepSeek-V3 解析 → 返回结构化 JSON
  - 触发条件：规则置信度 < 0.7 或 必填字段缺失 或 JD 极端短(<50字)/长(>5000字)

任务 5 — 三层解析编排逻辑
  - 规则解析 → 置信度检查 → (不足) LLM解析 → 返回结果
  - 缓存逻辑：同 URL hash 不重复解析

任务 6 — 技能词表
  - 构建初始技能词表（直接相关 vs 通用技能）
  - 正则 + 词表匹配提取技能
  - 权重标记（×2.0 / ×0.5）

任务 7 — JD 生命周期
  - active（默认）/ expiring_soon（30天）/ expired（停招）
  - 重抓覆盖时保留旧快照到 jd_parse_snapshots

【实现时请使用 Workflow 并行处理独立任务。先出 Plan 确认。】

```

  - 软性：项目相关性/行业重合度/职责语义相似度

任务 2 — 匹配度展示 UI

  - 技能缺口雷达图
  - 缺失技能列表（红色标记）
  - 匹配技能列表（绿色标记）

任务 3 — AI 简历优化（核心）

  - 输入：原始简历 + JD 解析结果 + 用户选择的改动模式
  - 输出：修改版简历 + 改动列表（每处改动含：位置/原文/新文/颜色/说明）

  - 🟡 侧重点调整 → 自动模式直接应用

  - "继续优化"/"就这样"/"考虑放弃"
