# EveryDeliver — 求职者半自动投递辅助平台

> **核心定位**：通过 AI 智能筛选与适配提升投递效率
> **当前阶段**：MVP 设计锁版，准备进入开发

---

## 解决的痛点

1. **多平台搜寻匹配岗位 + 简历定制 + 投递耗时巨大**：在官网、公众号、BOSS 直聘、小红书等平台中搜寻匹配岗位，同时准备简历、投递、面试，耗费大量时间
2. **不确定简历与岗位的匹配度**：想知道简历与 JD 的匹配程度，以及如何优化才能提升通过初筛的命中率

---

| **H. 多端架构** | 桌面端(Tauri) + 浏览器插件(Chrome/Edge) + 微信小程序 + 云端同步 |

## MVP 范围

### P0 必做（第一期）
1. 简历上传 + 解析（PDF/Word + 纯文本）
2. 求职偏好（白/黑名单，含 4 种自动检测机制）
3. 手动导入 JD + 浏览器插件抓取（BOSS 直聘 + 猎聘 + 通用）
4. JD Parser 完整实现（三层解析 + 生命周期）
5. 匹配度评估（硬性 60% + 软性 30% + 隐性 10%）
6. 简历 AI 优化（询问/自动双模式 + 改动高亮三色标记）
7. 简历库（多版本 + 模块化 + 投递快照）
8. 投递跳转 + 看板（三视图 + 7 状态 + 防重复 + 8 类提醒）
9. 多端基础架构（云端同步 + 账号体系 + 桌面端独享编辑权）

### P1（验证后）
- 浏览器插件表单自动填充
- 微信小程序
- 面试问题预测与准备清单
- 多模式优化深度打磨

### P2（规模化）
- 邮件解析自动状态跟踪
- 投递分析报告
- 社区标签库（黑名单众包）
- 工商信息 API 集成

## 关键设计决策

- ✅ **砍掉爬虫**：改用浏览器插件借助用户登录态抓取 JD
- ✅ **砍掉一键投递**：改用浏览器插件表单自动填充 + 用户手动提交
- ✅ **不存密码**：不存储任何第三方平台账号密码 / cookies
- ✅ **隐私分级**：敏感字段本地加密，通用字段才送 LLM
- ✅ **桌面端独享编辑权**：小程序和浏览器插件只读 + 同步
- ✅ **国产模型**：DeepSeek-V3 + Qwen 系列，成本约 ¥0.86/月/用户

---

## 已砍掉的功能

| 功能 | 原因 |
|------|------|
| 自动搜索接口（爬虫） | 法律风险 + 维护成本 |
| 一键投递 | 违反平台协议 + 封号风险 |
| 信任度可视化 | 用户不需要的指标 |
| "是否做过"二次确认 | 改为面试准备赋能 |
